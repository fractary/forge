# Mapping: claude-code → pi (hooks)

This file defines all rules for converting Claude Code hook files (`hooks/`) to pi.dev extension stubs (`extensions/*.ts`).

**Full automation is NOT possible for this conversion direction.** Shell scripts cannot be automatically ported to TypeScript. The converter generates TypeScript stub files with the original logic preserved as comments and TODO markers.

---

## Source Structure

- **Location**: `hooks/` directory
- **Config file**: `hooks/*.json` — defines hook events and which scripts to run
- **Script files**: `hooks/*.sh` — shell scripts executed by the hooks
- **Hook events**: `PreToolUse`, `PostToolUse`, `Stop`, `SubagentStop`, `PreCompact`, `Notification`, etc.

**Example source config (`hooks/pre-tool-use.json`):**
```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [{ "type": "command", "command": "hooks/validate-bash.sh" }]
      }
    ]
  }
}
```

**Example source script (`hooks/validate-bash.sh`):**
```bash
#!/bin/bash
echo "Validating bash command..."
# ... logic
```

---

## Target Structure

- **Location**: `extensions/{event-name}.ts`
- **Format**: TypeScript file implementing pi's `IExporter` extension interface
- **Content**: Stub with original shell logic preserved as a comment block

**Example target (`extensions/pre-tool-use.ts`):**
```typescript
// Pi extension stub — generated by fractary-forge harness converter
// Original source: hooks/pre-tool-use.json + hooks/validate-bash.sh
//
```

## Field Mapping Table

| Source Component | Target Component | Notes |
|---|---|---|
| Hook event name (e.g., `PreToolUse`) | Filename (e.g., `pre-tool-use.ts`) | camelCase → kebab-case |
| Shell script content | Comment block in stub | Preserved verbatim as `// ` prefixed lines |
| Hook matcher (e.g., `"Bash"`) | Comment in stub | Document which tools triggered the hook |

---

## Naming Convention Transforms

Hook event name → TypeScript filename:
```
PreToolUse     → pre-tool-use.ts
PostToolUse    → post-tool-use.ts
Stop           → stop.ts
SubagentStop   → subagent-stop.ts
PreCompact     → pre-compact.ts
Notification   → notification.ts
```

Rule: Split on camelCase boundaries, lowercase, join with `-`, append `.ts`.

## Output Location

```
{output_dir}/extensions/{event-name}.ts
```

Create the `extensions/` directory before writing.

---

## Conversion Approach

**Stub** — full automation is NOT possible. Shell scripts must be manually ported to TypeScript.

All hook conversions MUST:
1. Set `manual_review_required: true` in the result
2. Include original source content as a comment block in the stub
3. Include a TODO marker and reference to pi extension documentation

---

## Unsupported Features

1. **Shell script logic**: Cannot be automatically translated to TypeScript — requires manual implementation
2. **Hook matchers**: Claude uses matchers (e.g., `"Bash"`) to filter which tools trigger the hook; pi's extension API may differ — review required
3. **Environment variables**: Shell scripts may reference env vars that need to be handled differently in TypeScript
4. **CLI calls in hooks**: `fractary-core` CLI calls in shell scripts must be replaced with SDK calls or kept as `child_process` execs in TypeScript

---

## Validation Checklist

- [ ] One `.ts` stub file created per hook event
- [ ] Original JSON config preserved as comment block
- [ ] Original shell script content preserved as comment block
- [ ] TODO markers present
- [ ] Export function name matches event name (camelCase)
- [ ] `manual_review_required: true` in result
- [ ] Output at `extensions/{event-name}.ts`
